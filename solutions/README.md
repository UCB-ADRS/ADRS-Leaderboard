# Solutions Directory

This directory contains solutions for each problem in the ADRS Leaderboard. Solutions are organized by problem name and solution type.

## Solution Types

### Framework-Evolved Solutions

Solutions generated by AI-driven research frameworks:

- **`gepa/`** - Solutions evolved using [GEPA](https://github.com/ucbskyadrs/gepa) (Generative Evolutionary Program Architecture)
- **`openevolve/`** - Solutions evolved using [OpenEvolve](https://github.com/ucbskyadrs/openevolve)
- **`shinkaevolve/`** - Solutions evolved using [ShinkaEvolve](https://github.com/SakanaAI/ShinkaEvolve/tree/main)

Framework-evolved solutions were generated with GPT-5 using default configuration over 100 iterations.

### Baseline Solutions

- **`baseline/`** - Baseline reference implementations for comparison

### Human State-of-the-Art (SOTA)

- **`human_SOTA/`** - Human-written state-of-the-art solutions representing best known performance

## Directory Structure

```
solutions/
  {problem_name}/
    {solution_type}/
      resources/
        solution.py    # Solution implementation
      solve.sh         # Script to stage solution
```

## Status

⚠️ **Work in Progress** - This directory is under active development. Some solution folders are still missing for certain problems.

## Solution Requirements

Each solution must implement:
- `resources/solution.py`: A `Solution` class with a `solve()` method
- `solve.sh`: Script to stage the solution code into the execution environment

See the main [README.md](../README.md) for more details on solution structure and submission guidelines.
